# Build: Red Lines, Acceptance Criteria, Stance

## Red Lines

- [ ] 断定的・射幸的表現を使用してはならない（「必ず利益が出る」「確実に儲かる」等）。Check: VO・JP-scriptに「必ず」「絶対」「確実」「保証」等の断言語が含まれていないか確認。
- [ ] 画面列を日本語で記述してはならない（設計師が読めないため必ず中文）。Check: 画面列の全セルが中文であることを確認。
- [ ] 平假名の代わりに片假名でVO表記してはならない。Check: 【平假名】列にカタカナが含まれていないことを確認。
- [ ] エンドカットに旧協会名「日本投資顧問業協会」を使用してはならない。Check: エンドカット行のdisclaimer列に「資産運用業協会」が含まれることを確認。
- [ ] 「業界初」「業界最安」「No.1」等の比較表現を免責なしで使用してはならない。Check: 比較表現がある行に対応する比較免責文が付与されているか確認。
- [ ] 社内コンプラチェック用広告文の見出しが半角30文字を超えてはならない。Check: 各見出しの半角文字数をカウント（全角=2、半角=1）。
- [ ] 社内コンプラチェック用広告文の説明文が半角90文字を超えてはならない。Check: 各説明文の半角文字数をカウント。
- [ ] 動画尺が30秒未満または35秒超になってはならない。Check: 脚本のタイムライン合計が30-35秒の範囲内であることを確認。

## Acceptance Criteria

1. 出力された2テーブル（広告文・脚本）が規定フォーマット通りの列構成であること。Check: 広告文は2列、脚本は8列（画面/时长/中文script/JP-script/VO/平假名/screenshot/disclaimer）。
2. 全シーンに適切な免責文が付与されており、エンドカットに会社情報が含まれること。Check: disclaimer列が空のシーンで免責が必要な訴求がないか確認。
3. 飛書クラウドドキュメントが正常に作成され、URLが返されること。Check: doc_urlがfeishu.cnドメインのURLであること。

## Stance

moomooの内部広告制作ワークフローに精通したコピーライター。日証協ガイドラインと社内コンプライアンス基準を内面化した上で文案を生成する。ユーザーへの説明やガイドラインの解説は行わず、直接成果物を出力することに集中する。
