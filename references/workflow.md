# Workflow: moomoo Video Script Generation

## Phase 1 — 入力収集

Use AskUserQuestion with 3 questions simultaneously:

```
Q1: 広告投放渠道
  Options: Google/YouTube, Yahoo!, YouTube単独, その他
  (複数選択可)

Q2: 投放目标人群
  Options: 米株進階/高頻取引層, 米株初心者, 長期投資家（安定志向）, その他
  (単一選択)

Q3: 行情背景および故事線
  Options: 自由入力（市場背景・訴求したい機能・ストーリーの方向性を入力）
  Free text
```

受け取った3つの入力を確認してから Phase 2 に進む。

---

## Phase 2 — コンテンツ設計

### 2-1. テーマ・USP選定

背景/故事線から以下を決定する：
- 主軸USP（例：24時間取引・AI分析・手数料・決算機能等）
- 訴求する機能一覧（最大3つ）
- シーン数と秒数配分（合計30-35秒）

### 2-2. シーン構成（標準テンプレート）

| シーン | 目的 | 秒数目安 |
|---|---|---|
| Hook | 視聴者の興味を掴む。数字/問いかけ/痛点提示 | 0〜5s |
| 課題提起 | ターゲットの悩みを言語化 | 5〜9s |
| ブリッジ | moomooなら解決できると示す | 9〜12s |
| 機能紹介1 | 主軸機能のデモ/説明 | 12〜17s |
| 機能紹介2 | サブ機能（あれば） | 17〜22s |
| 機能紹介3 | サブ機能（あれば） | 22〜26s |
| CTA | 口座開設/アプリDLへの誘導 | 26〜29s |
| エンドカット | 会社情報 | 29〜33s |

シーン数は内容に合わせて増減可。合計秒数を必ず30-35秒に収める。

## Phase 3 — 脚本生成

### 3-1. 各列の生成ルール

**画面（中文）**
- 中文のみ。日本語禁止
- 具体的な映像指示（カメラアングル・アニメーション・テキスト表示等）

**中文 script**
- 各シーンの訴求ポイントを中文で簡潔に記述（制作チーム向け）

**JP-script（画面展示文案）**
- 画面に表示される日本語テキスト
- 断定的・射幸的表現禁止
- 簡潔でインパクトのある表現

**VO（旁白文案）**
- ナレーション原稿
- 話し言葉・自然なリズム
- JP-scriptと完全一致でなくてよい（補足・拡張可）

**【平假名】**
- VOを平假名で表記（録音ガイド用）
- 片假名は使用禁止
- 句読点・記号はそのまま

### 3-2. コンプライアンスチェック（生成後に必ず実行）

各シーンに対して以下を確認：
1. 画面列→中文のみか
2. 平假名列→カタカナ（ア-ン）が混入していないか
3. VO/JP-script→「必ず」「絶対」「確実」「保証」等の断言語がないか
4. 比較表現（「業界初」「業界最安」「No.1」）→disclaimer列に比較免責があるか
5. 画面に企業ロゴ・銘柄ロゴ・第三者商標が登場するシーン→第三者ロゴ免責があるか
6. moomooの機能・サービスを紹介するシーン→「口座開設などの条件」免責があるか

違反があれば修正してから次のPhaseへ。

## Phase 4 — 社内コンプラチェック用広告文生成

### 4-1. 見出し（半角30以内）× 2バリエーション

生成後に必ず文字数カウント：全角=2、半角=1。超過なら短縮。
**文字数カウント結果はテーブルセルに含めない。内部チェックのみ。**

例：「決算シーズン、準備はできてる？」= 28半角 → 上限内 → セルには文字列のみ掲載

### 4-2. 説明文（半角90以内）× 各見出しに対応

生成後に必ず文字数カウント。超過なら短縮。
**文字数カウント結果はテーブルセルに含めない。内部チェックのみ。**

### 4-3. テーブル形式

```
| 見出し｜文字数(半角30) | 説明文（半角90） |
|---|---|
| [見出し1テキストのみ] | [説明文1テキストのみ] |
| [見出し2テキストのみ] | [説明文2テキストのみ] |
```

## Phase 6 — 飛書ドキュメント出力

### 6-1. Markdownを構築

以下の構造で lark-table 形式のMarkdownを構築する：

```markdown
## 社内コンプラチェック用広告文

<lark-table column-widths="200,530" header-row="true">
[広告文テーブル]
</lark-table>

---

## 脚本 | [尺]秒 | [渠道] | [人群]

<lark-table column-widths="130,50,80,100,100,100,50,120" header-row="true">
[脚本テーブル 8列]
</lark-table>
```

脚本テーブル列順：画面（中文）/ 时长 / 中文 script / JP-script（画面展示文案）/ VO（旁白文案）/ 【平假名】/ screenshot / disclaimer

### 6-2. ドキュメント作成

Markdownファイルを `data/lark_draft_[テーマ].md` として保存し、以下のコマンドで飛書ドキュメントを作成する：

```bash
cd "<ファイルのあるディレクトリ>"
lark-cli docs +create --title "【Video Script】[テーマ篇]" --markdown "@lark_draft_[テーマ].md"
```

注意：`--markdown` フラグには `@ファイル名` 形式の相対パスを使用する（絶対パス不可）。必ずファイルと同じディレクトリに `cd` してから実行する。

作成完了後、doc_url をユーザーに返す。
